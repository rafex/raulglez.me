# PRODUCT.md

Fuente de verdad del producto.

## Debe responder

- Que problema se resuelve.
- Para quien se resuelve.
- Por que importa ahora.
- Que resultados espera el negocio o el usuario.
- Que queda fuera de alcance.

## Template

### Problema

Describe el problema principal en lenguaje simple.

### Usuarios

- Usuario o segmento:
  necesidad, dolor y contexto.

### Objetivos

- Objetivo principal
- Metricas de exito

### No objetivos

- Lo que este proyecto no intenta resolver.

### Valor diferencial

Explica por que esta solucion merece existir.
